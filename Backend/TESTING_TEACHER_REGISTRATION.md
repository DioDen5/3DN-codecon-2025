# Інструкція для тестування реєстрації викладачів

## Підготовка тестових даних

### 1. Створити Teacher профіль (створений адміном) для тестування входу з кодом:

```bash
cd Backend
node create-test-teacher.js
```

Це створить Teacher профіль з email `test.teacher@lnu.edu.ua` БЕЗ прив'язки до User акаунту.

### 2. Тестування різних сценаріїв:

## Сценарій 1: Teacher профіль вже існує (створений адміном)

**Очікуваний результат:** Показується форма входу з кодом

1. Відкрити форму реєстрації (Signup)
2. Обрати роль **"Викладач"**
3. Ввести email: `test.teacher@lnu.edu.ua`
4. Натиснути **"Перевірити email"**

**Що має статися:**
- Показується форма входу з кодом
- Код автоматично надсилається на пошту (у dev режимі він відображається в консолі backend)
- Повідомлення: "Профіль викладача з такою поштою вже існує. Підтвердіть вхід за кодом"

## Сценарій 2: User з таким email вже існує (наприклад, студент)

**Очікуваний результат:** Показується блок з опціями "Увійти" / "Ввести інший email"

1. Відкрити форму реєстрації
2. Обрати роль **"Викладач"**
3. Ввести email існуючого User (наприклад: `student11@lnu.edu.ua` з seed.js)
4. Натиснути **"Перевірити email"**

**Що має статися:**
- Показується помилка: "Користувач з таким email вже існує"
- Показується блок з двома кнопками:
  - "Увійти в систему" → перекидає на форму логіну
  - "Ввести інший email" → очищує поле email

## Сценарій 3: Email вільний (можна реєструватися)

**Очікуваний результат:** Можна продовжити реєстрацію

1. Відкрити форму реєстрації
2. Обрати роль **"Викладач"**
3. Ввести НОВИЙ email (наприклад: `new.teacher@lnu.edu.ua`)
4. Натиснути **"Перевірити email"**

**Що має статися:**
- Показується повідомлення: "Email вільний. Можна продовжувати реєстрацію."
- Показується кнопка **"Продовжити реєстрацію"**
- При натисканні відкривається багатокрокова форма реєстрації

## Тестові email адреси:

- `test.teacher@lnu.edu.ua` - Teacher профіль створений адміном (без User)
- `student11@lnu.edu.ua` - Існуючий User (студент)
- `teacher@lnu.edu.ua` - Існуючий User (викладач)
- `new.teacher@lnu.edu.ua` - Новий email для тестування реєстрації

## Де знайти код верифікації (у dev режимі):

Якщо backend працює в dev режимі, код верифікації відображається в консолі backend:
```
=== VERIFICATION CODE EMAIL (DEV MODE) ===
To: test.teacher@lnu.edu.ua
Type: login
Code: 123456
==========================================
```

Якщо потрібно видалити тестовий Teacher профіль:

```javascript
// В MongoDB або через скрипт

```
